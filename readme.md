# Webmanka - E-commerce Personalizado (WordPress & WooCommerce)

Este repositorio contiene el desarrollo y las personalizaciones técnicas realizadas para el sitio web **[www.webmanka.com](https://www.webmanka.com)**. El proyecto se enfoca en una experiencia de usuario optimizada, integración de pagos regionales y automatización de marketing.

## 🚀 Resumen del Proyecto

Webmanka es una plataforma de comercio electrónico construida sobre **WordPress**, utilizando el framework de **Astra** como base. Mi trabajo consistió en la arquitectura de la tienda, la modificación del código fuente para ajustar el diseño a la identidad de marca y la integración de un ecosistema completo de plugins de venta y SEO.

## 🛠️ Tecnologías y Herramientas

* **Lenguajes:** * **PHP:** Lógica de servidor y hooks personalizados para WooCommerce.
    * **JavaScript:** Interactividad en el frontend y optimización de carga.
    * **CSS3:** Estilos personalizados (Custom CSS) para modificar el layout original de Astra.
* **CMS:** WordPress.
* **E-commerce:** WooCommerce.
* **Infraestructura:** Hostinger (Alojamiento y gestión de DNS).

## 🔧 Implementaciones Técnicas Destacadas

### 1. Pasarela de Pagos (Mercado Pago)
Configuración e integración de la API de **Mercado Pago** para procesar transacciones seguras, permitiendo pagos con tarjetas de crédito, débito y efectivo local, garantizando una alta tasa de conversión en el checkout.

### 2. Optimización SEO (Yoast SEO)
Implementación de estructuras de datos y metadatos mediante **Yoast SEO** para mejorar el posicionamiento orgánico en buscadores, optimizando el indexado de productos y categorías.

### 3. Ecosistema Social & Comunicación
* **WhatsApp Business:** Integración de botón de chat directo para soporte al cliente en tiempo real.
* **Social Connect:** Sincronización de catálogo con **Facebook Shop** e **Instagram Shopping** para ventas multicanal desde WooCommerce.

### 4. Personalización del Tema (Astra Modifications)
En lugar de usar el tema "out of the box", realicé modificaciones mediante:
* **Functions.php:** Snippets personalizados para alterar el comportamiento de WooCommerce (ej. cambio de etiquetas, flujos de envío).
* **Child Theme:** Uso de una estructura de tema hijo para asegurar la persistencia de los cambios ante actualizaciones del núcleo.

## 📂 Estructura del Repositorio

Para mantener este repositorio profesional, solo he incluido los archivos donde mi intervención de código fue directa:
* `/css`: Hojas de estilo personalizadas.
* `/js`: Scripts de interactividad.
* `functions.php`: Lógica personalizada en PHP.
* `README.md`: Documentación del proyecto.
